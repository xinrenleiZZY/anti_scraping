项目结构总览

amazon_scraper_system/
├── docker-compose.yml          # 三个服务：postgres, backend, frontend
├── init.sql                    # 数据库初始化（自动执行）
├── .env                        # 环境变量配置
├── scraper_config.json         # 爬虫关键词配置（手动改这里）
│
├── backend/
│   ├── Dockerfile
│   ├── requirements.txt
│   ├── main.py                 # FastAPI 入口，注册路由
│   └── app/
│       ├── config.py           # 全局配置（读 .env）
│       ├── database.py         # 数据库连接
│       ├── models.py           # SQLAlchemy 表模型
│       ├── schemas.py          # Pydantic 响应模型
│       ├── crud.py             # 数据库查询操作
│       ├── api/
│       │   ├── data.py         # GET /api/results, /api/tasks
│       │   └── scraping.py     # POST /api/scrape（触发爬虫）
│       └── scraper/
│           ├── scraper_config.json      # 关键词/页数/邮编配置
│           ├── auto_amazon_scraper.py   # 核心爬虫
│           ├── dataprocess.py           # 预处理（加字段）
│           ├── pipeline.py              # 串联爬取→预处理→入库
│           ├── data_processor.py        # 数据格式转换
│           ├── headers_manager.py       # 请求头管理
│           ├── anti_scraping_config.py  # 反爬配置
│           ├── amazon_data/             # 爬虫原始输出
│           └── processed_data/          # 预处理后输出
│
└── frontend/
    ├── index.html
    ├── style.css
    └── app.js
数据流

scraper_config.json
       ↓
auto_amazon_scraper.py → amazon_data/*.json
       ↓
dataprocess.py         → processed_data/*_processed.json
       ↓
pipeline._save_to_database() → PostgreSQL
API 端点
方法	路径	说明
GET	/	健康检查
GET	/api/results	查询爬取结果，支持 ?keyword=
GET	/api/tasks	查询任务列表
POST	/api/scrape	触发爬取，支持 ?keyword=&pages=
手动运行流程

# 1. 改关键词
# 编辑 backend/app/scraper/scraper_config.json

# 2. 爬取
cd backend/app/scraper
python auto_amazon_scraper.py

# 3. 预处理
python dataprocess.py

# 4. 入库（通过 API）
curl -X POST http://localhost:8000/api/scrape
