一、需求背景
当前需要人工在亚马逊前台搜索关键词，查看某个ASIN的自然排名和广告排名，存在问题：
1.操作耗时，无法批量处理
2.数据不连续，无法做趋势分析
3.难以长期监控关键词表现

二、需求目标
开发一个工具，实现：
1.输入美国邮编+ASIN + 关键词 → 自动返回该ASIN在搜索结果中的自然排名和广告排名
2.支持每日自动抓取，用于趋势分析，可以抓取并保存页面截图
3.抓取结果需尽量与人工搜索结果一致（允许合理误差范围，见验收标准）

三、核心功能
1. 关键词排名抓取
输入：ASIN、关键词、站点（如 US）、设备（PC ）
抓取逻辑：
自动打开亚马逊搜索页面（https://www.amazon.com/s?k=关键词）   爬取
清除浏览器缓存和 cookies，使用隐私模式/未登录状态  X
模拟普通用户访问（避免频繁刷新/间隔抓取，每次访问延迟 2-5 秒）  机器人
PC 端抓取
抓取前 10 页（如果关键词搜索结果不足 10 页，抓到最后一页即可）  page=10

2. 自然排名（Organic Rank）
定义：页面中不带“Sponsored”标识的商品（全量爬取，后续数据清洗排除）
抓取规则：按出现顺序排序，所有广告位必须剔除后再计算排名
返回字段：自然排名，所在页数
示例：第1个是广告（不算）
第2个是自然 → 排名=1
第3个是广告（不算）
第4个是自然 → 排名=2

3. 广告排名（Sponsored Rank）
定义：只要带“Sponsored”标识，均为广告位
抓取规则：同一个 ASIN 可能出现多个广告位，必须全部记录
广告类型：
SP（Sponsored Products）：含 “Sponsored” 标签，且为单商品卡片
SB（Sponsored Brands）：页面顶部横幅，多商品组合
SB Video（视频广告）：视频播放器模块
返回字段：广告类型（SP / SB / SB Video），所在页数，广告排名
4. 多位置记录
同一个ASIN可能：同时有自然排名 + 广告排名、同一页出现多个广告位
要求：必须全部记录，不允许只保留一个

四、设备和翻页规则
PC：按分页抓取，最多前 10 页
Mobile：按分页或屏幕滚动抓取，最多前 10 屏
同页抓取：必须抓取页面所有广告与自然排名
如未抓取到对应数据，需返回空值或标记为 null

五、输入与输出数据格式
输入方式：

手动输入：美国邮编+ASIN + 关键词       
批量上传：Excel（支持多 ASIN、多关键词）


输出字段（每条记录一行）：
字段	说明
date	抓取日期    
keyword	关键词
asin	ASIN
marketplace	站点
device	PC / Mobile
organic_rank	自然排名
organic_page	自然页
organic_placement	具体位置
例：第2页第3位
ad_type	广告类型（SP / SB / SB Video）
ad_rank	广告排名
ad_page	广告页
ad_placement	具体位置
例：第1页第5位
		
六、异常情况处理

情况	返回值
未找到	ASIN Not Found
抓取失败	Error
排名异常波动	返回昨日 vs 今日排名差值（可自定义阈值，例如： ±3 名以内为正常，运营自己定义）
抓取失败处理：
如果人工搜索可见，但抓取为空 → 判定为 Error
系统可自动重试 2 次，如果仍失败 → 标记 Error
正常情况：
ASIN 只存在自然排名 → 正常
ASIN 只存在广告排名 → 正常
同时存在自然排名和广告排名 → 正常

七、验收标准
验证方法：人工在亚马逊搜索关键词，找到该ASIN位置，对比抓取数据
验收要求：
自然排名允许 ±1 名误差，广告排名允许 ±1 个广告位误差；页数尽量一致，如因刷新差异跨页 ±1 页可接受
如果人工搜索 ASIN 可见，但系统抓取为空 → 判定为抓取异常
ASIN 可能只存在自然排名，广告排名 或者两者同时存在→ 正常，不算异常
验收建议在同一时间段内抓取和人工比对，减少刷新导致的差异

八、抓取频率
支持：手动抓取、每日抓取（常用）、每周抓取

九、补充说明
Amazon页面结构可能变化，需具备一定适配能力
抓取需避免被识别为异常访问（如频繁请求）
建议固定地区环境（避免结果波动）
